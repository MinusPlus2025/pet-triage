# 毛孩急诊官 · Pet Triage

> 本项目为比赛原型，用于展示宠物症状分级流程，不构成疾病诊断或专业兽医建议。紧急情况下，请直接联系宠物医院。
>
> 当前版本采用前端直连模型 API 的方式，仅适合本地比赛演示。请勿在公开仓库、前端代码或 Git 提交记录中放入真实 API Key。

拍照、描述症状，30 秒判断宠物现在要不要去医院。

产品不尝试替代兽医诊断，只回答一个问题：

**现在要不要去医院？**

系统输出三级就医建议：

- **绿色：可以在家观察**
- **黄色：这两天带它去看看**
- **红色：现在就走**

---

## 产品定位

宠物出现异常时，主人往往不知道问题是否严重。

尤其是在深夜、独处或附近医院关闭的情况下，用户需要的不是复杂的疾病百科，而是一个清晰、直接的行动建议。

毛孩急诊官通过宠物种类、年龄、症状描述和照片等信息，帮助用户快速判断当前情况的紧急程度。

本产品：

- 不提供疾病确诊
- 不提供处方或药物剂量
- 不代替执业兽医
- 不建议用户延误紧急就医
- 只提供初步的就医紧急程度参考

---

## 核心功能

### 症状输入

用户可以填写：

- 宠物种类
- 宠物年龄
- 症状描述
- 症状持续时间
- 精神和食欲状态
- 相关照片

### 智能追问

当信息不足时，系统会围绕以下信息进行追问：

- 症状持续多久
- 出现频率
- 是否进食
- 精神状态是否正常
- 是否存在呼吸、排尿、抽搐等危险信号

### 三级分诊结果

#### 绿色 · 可以在家观察

适用于暂时没有明显危险信号、精神食欲正常、症状较轻的情况。

系统会给出：

- 判断依据
- 居家观察建议
- 需要升级就医的异常信号

#### 黄色 · 这两天带它去看看

适用于暂时不属于立即急诊，但症状持续存在或需要专业检查的情况。

系统会给出：

- 判断依据
- 近期就诊建议
- 就诊前需要记录的信息
- 可以向医院咨询的检查方向

#### 红色 · 现在就走

适用于存在明显危险信号、延误可能加重风险的情况。

系统会给出：

- 判断依据
- 为什么不能等待
- 立即行动建议
- 查找附近宠物医院的入口

- Vite

## 本地运行

### 1. 安装依赖

```bash
npm install
```

### 2. 创建环境变量文件

复制环境变量模板：

```bash
cp .env.example .env
```

### 3. 配置模型

默认使用 Qwen（支持图片）：

```env
VITE_MODEL_PROVIDER=qwen
VITE_QWEN_API_KEY=你的比赛专用Key
```

也可以切换到 Gemini（支持图片）：

```env
VITE_MODEL_PROVIDER=gemini
VITE_GEMINI_API_KEY=你的比赛专用Key
```

或切换到 DeepSeek（纯文字）：

```env
VITE_MODEL_PROVIDER=deepseek
VITE_DEEPSEEK_API_KEY=你的比赛专用Key
```

### 4. 启动项目

```bash
npm run dev
```

浏览器打开终端中显示的本地地址即可。

## 环境变量模板

公开仓库中的 `.env.example` 只应保留空值：

```env
# Model provider: qwen / gemini / deepseek
VITE_MODEL_PROVIDER=qwen

# Keep all values empty in the public repository
VITE_QWEN_API_KEY=
VITE_GEMINI_API_KEY=
VITE_DEEPSEEK_API_KEY=
```

真实 Key 只保存在本地 `.env` 文件中。

不要将 `.env` 上传到 GitHub。

## 模型说明

### Qwen / Gemini（默认）

比赛 Demo 默认使用 Qwen 视觉模型，支持读取图片。

Qwen 和 Gemini 均为多模态模型：

- 症状文字和上传的照片会一起发送给模型
- 模型参考图片和文字共同判断

### DeepSeek

DeepSeek Chat 为纯文本模型，仅作为备用选项。

当模型提供方设置为 DeepSeek 时：

- 症状文字会发送给 DeepSeek
- 上传功能不可用，无法发送照片
- 结果主要依据用户的文字描述生成

因此，使用 DeepSeek 时，用户需要尽量完整地描述症状。

需要注意：

- 照片会发送给第三方模型服务
- 不要上传包含人脸、地址、联系方式等敏感信息的图片
- 图片只能作为初步参考，不能替代兽医检查

## 安全说明

当前比赛 Demo 采用前端直连模型接口的方式。

所有以 `VITE_` 开头的环境变量都会进入前端构建结果，因此：

- 不要在公开仓库中提交真实 API Key
- 不要将 Key 直接写入源代码
- 不要提交 `.env` 文件
- 建议使用单独创建的比赛专用 Key
- 建议限制比赛 Key 的额度和使用范围
- 比赛结束后及时撤销临时 Key
- 不要在正式生产环境中使用当前前端直连结构

正式上线时，应改为：

```text
浏览器
  ↓
自建后端或 Serverless API
  ↓
DeepSeek / Qwen / Gemini
```

API Key 应只保存在服务器环境变量中。

---

## 隐私说明

用户输入的症状描述可能会发送至第三方模型平台。

使用支持图片的模型时，用户上传的照片也可能发送至对应模型服务。

请勿上传包含以下内容的信息：

- 用户本人或家人的清晰人脸
- 家庭地址
- 手机号或联系方式
- 快递单
- 身份证件
- 银行信息
- 其他与宠物症状无关的隐私内容

当前比赛版本不建立用户账户，也不主动保存用户历史记录。

---

## 使用边界

本工具不能替代宠物医院和执业兽医。

模型输出可能受到以下因素影响：

- 用户描述不完整
- 用户描述存在误差
- 照片质量不足
- 模型理解偏差
- 网络或接口异常
- 模型输出格式错误

当系统无法可靠判断时，不应将异常结果自动降级为“可以等待”。

## 紧急情况提示

如宠物出现以下情况，请优先联系宠物医院或立即就医：

- 呼吸困难
- 猫出现张口呼吸
- 牙龈明显发白或发紫
- 长时间无法排尿
- 频繁蹲猫砂盆但没有尿
- 抽搐或意识异常
- 突然瘫痪
- 严重外伤
- 被车辆撞击
- 从高处坠落
- 疑似误食有毒物质
- 持续呕吐
- 呕吐物或排泄物明显带血
- 幼猫幼犬长时间拒食
- 高温环境下出现步态不稳或大量流涎

紧急情况不要只依赖模型结果。

## 项目声明

毛孩急诊官的目标不是告诉用户宠物得了什么病。

它只希望在用户最慌乱的时候，帮助用户更快地做出下一步决定：

**可以先观察、近期去看看，还是现在就走。**
