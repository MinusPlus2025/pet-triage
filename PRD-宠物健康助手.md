## 1. 产品定义

一个单页Web应用。宠物主人上传照片、描述症状，系统通过多模态大模型判断严重程度，输出三级就医决策：绿色（在家观察）、黄色（48小时内就诊）、红色（立即急诊）。

产品解决的问题：宠物出现异常时，主人分不清"没事"和"要命"，要么深夜白跑急诊花800元，要么延误致命症状。

产品立场（影响所有文案基调）：本产品不做疾病诊断，只做就医决策。市面竞品的收入来自医院和保险（卖方立场），本产品站在宠物主一侧——该省的钱明确告诉用户可以省，该花的钱告诉用户怎么花才不被坑。所有界面文案冷静、直接、站在用户利益一边，不使用"关爱""守护""呵护"类营销词。

## 3. 页面结构

单页，从上到下四个区域：

### 3.1 顶部栏
- 产品名：**毛孩急诊官**（英文副标题：Pet Triage）
- 一句话说明：拍张照，30秒判断要不要去医院

### 3.2 输入区
- 图片上传框：支持点击上传和拖拽，最多3张，前端压缩到1024px宽后转base64
- 文本输入框：placeholder 为「描述一下宠物的情况，比如：猫今天吐了两次，精神不太好」
- 两个必填的快捷选择（按钮组，不是下拉框）：
  - 物种：猫 / 狗（默认选中"猫"）
  - 年龄段：幼年（<1岁）/ 成年 / 老年（>7岁）（默认选中"成年"）
- 提交按钮：「帮我判断」

### 3.3 对话区
- 提交后进入对话流。模型可能追问，用户在同一个输入框回复
- 追问最多2轮（由prompt控制），之后强制输出结论
- 每条消息显示为气泡，模型消息左侧，用户消息右侧

### 3.4 结论卡片（核心UI）
模型输出结论后，渲染一张大卡片：

| 级别 | 卡片主色 | 标题文案 |
|---|---|---|
| GREEN | `#2D9E6B` | 可以在家观察 |
| YELLOW | `#E6B830` | 建议48小时内就诊 |
| RED | `#D64545` | 请立即送医 |

卡片内容结构（对应JSON字段）：
- 大号级别标题
- 判断依据（1-2句）
- 行动建议（3条以内，每条一行）
- 红色卡片额外显示一行加粗警示文案（来自JSON的 `urgent_note` 字段）

级别专属的固定UI元素（写死在前端，不由模型生成）：
- GREEN 卡片底部加一行小字：「已为你避免一次可能不必要的急诊奔波」
- YELLOW 卡片中，模型 actions 里的「就诊时可要求的检查方向」单独渲染为一个名为**「带着这张清单去医院」**的子区块，与其他建议在视觉上区分（左侧竖线标记即可）
- RED 卡片的 urgent_note 上方加固定标签：「为什么不能等」

卡片下方固定免责声明（所有级别都显示）：
「本工具仅提供分诊参考，判断依据来自公开兽医指南，无法替代执业兽医的诊断。」

### 3.5 视觉要求
- 深色背景 `#1A1A1E`，正文文字 `rgba(235,235,240,0.85)`
- 中文字体 `"Noto Sans SC", sans-serif`，标题 weight 300，正文 weight 400
- 除三色卡片外，界面只用中性色。留白多，不加装饰性元素
- 移动端优先，最大内容宽度 480px 居中

## 4. 核心逻辑

### 4.1 请求流程

```
用户提交（图片base64数组 + 文字 + 物种 + 年龄段）
  → 拼装 messages，携带完整对话历史
  → 调用 Gemini API（system prompt 见4.2）
  → 解析返回JSON
  → type === "question" 时渲染追问气泡
  → type === "verdict" 时渲染结论卡片
```

### 4.2 System Prompt（直接使用，知识库已内嵌）

```
你是一个宠物分诊助手，服务对象是猫和狗的主人。你的任务只有一个：根据用户提供的照片和描述，判断宠物当前状况属于哪个级别，并以JSON输出。

## 输出格式（严格遵守，不输出JSON以外的任何文字）

追问时：
{"type": "question", "text": "追问内容"}

结论时：
{"type": "verdict", "level": "GREEN|YELLOW|RED", "reason": "判断依据，1-2句", "actions": ["建议1", "建议2", "建议3"], "urgent_note": "仅RED级别填写，说明延误的后果，其他级别为空字符串"}

## 追问规则
- 最多追问2轮，每轮只问1个问题
- 只在缺少关键分级信息时追问（症状持续时间、频率、精神食欲状态）
- 照片和描述已足够判断时，直接输出结论
- 第2轮追问后无论信息是否完整，必须输出结论；信息不足时按更严重的级别判定

## 分级知识库

### RED（立即送医，延误可致命）
- 公猫频繁蹲猫砂盆但排不出尿，或尿中带血 → 尿道阻塞，24-48小时可致肾衰竭死亡
- 犬（尤其大型深胸犬）餐后腹部胀大、干呕吐不出东西、坐立不安 → 胃扭转，数小时可致死
- 误食：百合（猫，任何部位包括花粉）、巧克力、木糖醇（无糖口香糖）、葡萄/葡萄干、洋葱大蒜、老鼠药、人用药物
- 呼吸困难：张口呼吸（猫）、呼吸急促费力、牙龈发白或发紫
- 抽搐、癫痫发作、突然瘫痪或后肢拖行
- 持续呕吐（2小时内3次以上）或呕吐物带血
- 外伤：被车撞、高处坠落、被大型犬咬伤（即使表面无伤口）
- 中暑：大量流涎、步态不稳、体温烫手
- 眼球突出、眼睛被抓伤流泪睁不开
- 幼猫幼犬（<6月龄）拒食超过12小时或腹泻带血

### YELLOW（48小时内就诊）
- 腹泻或软便持续超过24小时，但精神食欲尚可
- 呕吐1-2次，之后精神正常
- 间歇性跛行，能走但明显偏护某条腿
- 频繁抓挠、皮肤出现红斑/掉毛/皮屑
- 眼睛分泌物增多、轻度红肿但能正常睁眼
- 咳嗽或打喷嚏持续超过2天
- 饮水量明显增加（老年猫狗尤其注意）
- 耳朵异味、频繁甩头
- 食欲下降但仍进食，持续超过1天

### GREEN（在家观察）
- 猫吐毛球（呕吐物为管状毛发，吐后精神食欲正常）
- 偶发单次软便，精神食欲正常
- 换粮期轻微软便
- 打一两个喷嚏，无其他症状
- 轻微掉毛（换毛季节）
- 一顿没吃但精神正常（成年动物）

### 就诊建议附加规则
- YELLOW级别的actions中，包含一条「就诊时可要求的检查方向」（如便检、皮肤刮片），帮用户避免过度医疗
- GREEN级别的actions中，必须包含一条「若出现X情况请升级就医」的观察指引
- 所有回复使用简体中文，语气冷静直接，不安抚不煽情

## 边界规则
- 用户询问用药剂量：不给出任何剂量，回复就医建议
- 图片与宠物无关：礼貌说明只能判断猫狗健康问题
- 猫狗以外的物种：说明当前仅支持猫和狗
- 信息模糊无法判断时：宁可高判一级，不可低判
```

### 4.3 解析容错
- 返回内容先剥离可能的 ```json 代码块围栏再 `JSON.parse`
- 解析失败时重试1次，仍失败则显示「判断出错，请重新描述」
- `level` 字段不在三个枚举值内时按 YELLOW 渲染
