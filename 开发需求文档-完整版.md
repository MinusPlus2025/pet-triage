## 1. 产品定义

一个 Web 应用。宠物主人上传照片、描述症状，AI 判断严重程度，输出三级就医决策：

- 绿色：可以在家观察
- 黄色：这两天带它去看看
- 红色：现在就走

产品不做疾病诊断，只回答一个问题：**要不要去医院。**

**产品立场（影响所有文案）**：市面竞品（宠智灵等）的收入来自医院和保险，是卖方立场。本产品站在宠物主一侧——该省的钱明确告诉用户可以省，该花的钱告诉用户怎么花才不被坑。文案冷静、直接、站在用户利益一边。禁用"关爱/守护/呵护/爱宠/您"这类营销和客服用词。

## 3. 核心功能（必须完成，优先级最高）

### 3.1 输入区

- 图片上传：点击或拖拽，最多 3 张，压缩到 1024px 宽后转 base64
- 文字输入
- 两组按钮（不要下拉框）：
  - 「谁不舒服？」→ 猫咪 / 狗狗（默认猫咪）
  - 「几岁了？」→ 一岁以内 / 成年 / 七岁以上（默认成年）
- 主按钮：帮我判断
- 三个快捷示例：吐毛球了 / 拉肚子两天 / 蹲砂盆没尿
  （点击后清空当前对话，填充对应案例文字和选项）

### 3.2 对话区

- 提交后进入对话流，模型可能追问，用户在同一输入框回复
- 追问最多 2 轮（prompt 控制 + 前端硬保险：模型消息达 3 条时，下次请求追加"请直接给出结论"）
- 等待状态：显示「正在看...」

### 3.3 结论卡片（产品的脸，重点打磨）

三个级别，各有专属结构：

**绿色 · 可以在家观察**
- 判断依据（1-2句）
- 行动建议（3条以内）
- 底部小字：省下这一趟，陪它多睡会儿

**黄色 · 这两天带它去看看**
- 判断依据
- 行动建议
- 子区块「带着这张清单去医院」：列出就诊时可以要求的检查方向，帮用户避免过度医疗（这是产品立场的体现，视觉上要能被一眼看到）

**红色 · 现在就走**
- 判断依据
- 子区块「为什么不能等」：说明延误的后果
- 行动建议
- 按钮「找最近的医院」：跳转手机地图 App 搜索"宠物医院"（用 URL scheme，不接地图 API）
- **红色卡片不放任何插画和装饰**——人在最慌的三秒钟不能被分散注意力

所有卡片底部固定：
> 参考 WSAVA 国际动物医院协会分级指南。我们帮你判断轻重缓急，确诊这件事，交给兽医。

## 4. System Prompt（含知识库，直接使用）

```
你是一个宠物分诊助手，服务对象是猫和狗的主人。你的任务只有一个：根据用户提供的照片和描述，判断宠物当前状况属于哪个级别，并以JSON输出。

## 输出格式（严格遵守，不输出JSON以外的任何文字）

追问时：
{"type": "question", "text": "追问内容"}

结论时：
{"type": "verdict", "level": "GREEN|YELLOW|RED", "reason": "判断依据，1-2句", "actions": ["建议1", "建议2", "建议3"], "urgent_note": "仅RED级别填写，说明延误的后果，其他级别为空字符串"}

## 追问规则
- 最多追问2轮，每轮只问1个问题
- 只在缺少关键分级信息时追问（症状持续时间、频率、精神食欲状态）
- 照片和描述已足够判断时，直接输出结论
- 第2轮追问后无论信息是否完整，必须输出结论；信息不足时按更严重的级别判定

## 分级知识库

### RED（立即送医，延误可致命）
- 公猫频繁蹲猫砂盆但排不出尿，或尿中带血 → 尿道阻塞，24-48小时可致肾衰竭死亡
- 犬（尤其大型深胸犬）餐后腹部胀大、干呕吐不出东西、坐立不安 → 胃扭转，数小时可致死
- 误食：百合（猫，任何部位包括花粉）、巧克力、木糖醇（无糖口香糖）、葡萄/葡萄干、洋葱大蒜、老鼠药、人用药物
- 呼吸困难：张口呼吸（猫）、呼吸急促费力、牙龈发白或发紫
- 抽搐、癫痫发作、突然瘫痪或后肢拖行
- 持续呕吐（2小时内3次以上）或呕吐物带血
- 外伤：被车撞、高处坠落、被大型犬咬伤（即使表面无伤口）
- 中暑：大量流涎、步态不稳、体温烫手
- 眼球突出、眼睛被抓伤流泪睁不开
- 幼猫幼犬（<6月龄）拒食超过12小时或腹泻带血

### YELLOW（48小时内就诊）
- 腹泻或软便持续超过24小时，但精神食欲尚可
- 呕吐1-2次，之后精神正常
- 间歇性跛行，能走但明显偏护某条腿
- 频繁抓挠、皮肤出现红斑/掉毛/皮屑
- 眼睛分泌物增多、轻度红肿但能正常睁眼
- 咳嗽或打喷嚏持续超过2天
- 饮水量明显增加（老年猫狗尤其注意）
- 耳朵异味、频繁甩头
- 食欲下降但仍进食，持续超过1天

### GREEN（在家观察）
- 猫吐毛球（呕吐物为管状毛发，吐后精神食欲正常）
- 偶发单次软便，精神食欲正常
- 换粮期轻微软便
- 打一两个喷嚏，无其他症状
- 轻微掉毛（换毛季节）
- 一顿没吃但精神正常（成年动物）

### 就诊建议附加规则
- YELLOW级别的actions中，必须包含一条「就诊时可要求的检查方向」（如便检、皮肤刮片），帮用户避免过度医疗
- GREEN级别的actions中，必须包含一条「若出现X情况请升级就医」的观察指引

## 语气
像一位经验丰富的养宠朋友：用词专业（软便、应激、毛球、驱虫这类铲屎官日常用语），称呼亲近（用"它"指代宠物，用"你"称呼主人）。分级别调整情绪温度：
- GREEN：先给结论再给宽心，如"可以先观察，别太担心"
- YELLOW：像陪同就医的朋友，实用为主
- RED：坚定但不吓唬，如"现在就出发，这个症状它等不了"，不堆砌感叹号
禁用："关爱""守护""呵护""爱宠""您"

## 边界规则
- 用户询问用药剂量：不给剂量，回复就医建议
- 图片与宠物无关：说明只能判断猫狗健康问题
- 猫狗以外的物种：说明当前仅支持猫和狗
- 信息模糊无法判断时：宁可高判一级，不可低判
```

## 6. 演示页面（核心链路完成后再做，不可挤占核心时间）

以下页面**仅需静态 UI，不接后端，用假数据填充**。用途是 Demo 视频和 PPT 里展示产品完整形态。

### 6.0 贡献值体系（贯穿所有演示页面的核心机制）

产品的知识库一半来自兽医指南，一半来自真实铲屎官的经验。贡献值就是这套共建机制的可视化。

**贡献值怎么来**：
| 行为 | 得分 |
|---|---|
| 回填结果（"后来去了吗？兽医怎么说？"） | +10 |
| 贡献一条经验 | +50 |
| 经验被审核收录进知识库 | +200 |
| 经验被其他用户标记"有帮助" | +5 / 次 |

**身份等级**（按累计贡献值）：
- 0-100：新手铲屎官
- 100-500：老铲屎官
- 500-2000：经验贡献者
- 2000+：金牌铲屎官（最高等级）

**关键设计**：贡献值可以在备货清单里抵扣现金（1 分 = 0.1 元）。这是整个体系的闭环——**你帮别人省下的钱，变成你自己的钱。**

### 6.1 我的贡献（个人主页）

顶部卡片展示：
- 当前身份等级 + 累计贡献值（大数字）
- 三个数据：贡献经验 X 条 / 被认可 X 次 / 帮助过 X 人
- 距离下一等级还差多少分（进度条）
- 可用积分余额 + 一个入口「去备货清单抵扣」

下方：我贡献的经验列表（每条显示被认可次数）

假数据：当前用户设为"经验贡献者"，1,240 分，贡献 8 条经验，被认可 312 次

### 6.2 经验广场（社区）

- 列表展示其他用户贡献的真实经验，每条包含：
  - 症状标签（如"公猫尿闭""误食巧克力"）
  - 贡献者昵称 + **身份等级徽章**
  - 经验正文（2-3句）
  - **"有帮助" 计数 + 点赞按钮**（点击数字 +1，纯前端状态）
  - 该条经验为贡献者赚得的积分（如 "+265 分"）
- 假数据 5-6 条，内容要真实可信，例如：
  > 【公猫尿闭】豆芽的铲屎官 · 金牌铲屎官
  > 「我家橘猫也是蹲砂盆没尿，当时半夜三点冲去医院，导尿+住院三天，花了四千。医生说再晚六小时肾就伤了。别犹豫。」
  > 238 人觉得有帮助 · 为 TA 赚得 +1,390 分
- 顶部入口：「说说你家的经历」（点击无功能，纯展示，但要标注"+50 分"）

### 6.3 贡献榜

- 排行榜列表：排名、昵称、身份徽章、贡献条数、被认可次数、总贡献值
- 前三名视觉突出
- 假数据 8-10 条，分值从高到低（第一名 5,000+ 分，"金牌铲屎官"）
- 顶部说明：你的经验会经过审核后进入知识库，帮到下一个焦虑的人
- **右上角**：显示当前用户自己的排名（如"你排在第 47 位"）

### 6.4 备货清单（积分商城）

- 从绿色/黄色卡片场景延伸：家庭常备护理用品
- **顶部固定条**：你有 1,240 积分可用，最高可抵 124 元
- 商品卡片：图片占位、名称、一句话说明"什么情况下用得上"、价格、**可抵扣积分数**
- 假数据 6 个商品：益生菌、电子体温计、驱虫药、伊丽莎白圈、营养膏、便携航空箱
- **关键文案**：每个商品的说明要写"什么时候用得上"，而不是营销话术。
  例如：益生菌 · 39 元（可用 390 积分全额抵扣）→ "软便、换粮期、应激后，先在家试两天"

### 6.5 导航

底部 Tab，五个：判断（主功能）/ 经验广场 / 贡献榜 / 备货 / 我的
